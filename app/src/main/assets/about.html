Intro

  • The app is developed using MVP architecture.
  • Room persistence library is used to save data in a local database. To fetch and update cashed data we have used Coroutines to perform background operations.
  • Also, in order to read CSV data we have used Coroutines to perform background operations.
  • The app is developed based on TDD with a proper test coverage.

Features:

Home screen

  • Shows a list of users.
  • If there are no cached data we will read User data from CSV file otherwise we will show cashed data.
  • When we pull down on a list of users (Swipe-to-refresh gesture) we will read user data from CSV without blocking UI thread.

Orientation

  • The application handles phone rotation (Portrait and Landscape).

Tests

  • We have implemented some Unit tests
  • In order to test database implementation we wrote a JUnit test (Instrumented unit tests)

About Screen

  • About screen is implemented using web view
