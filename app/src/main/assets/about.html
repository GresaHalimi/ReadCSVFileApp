Intro

  • The app is developed using MVP architecture.
  • UI is implemented using one activity and multiple fragments.
  • Room persistence library is used to save data in a local database.
  • In order to read CSV data we have used AsyncTask to perform background operations.
  • The app is developed based on TDD with a proper test coverage.

Features:

Home screen

  • Shows a list of users.
  • If there are no cached data we will read User data from CSV file otherwise we will show cashed data.
  • When we pull down on a list of users (Swipe-to-refresh gesture) we read CSV file using Async task without blocking UI thread.

Orientation

  • The application handles phone rotation (Portrait and Landscape).

Tests

  • In order to test User list we have implemented a Unit test
  • In order to test database implementation we have write a JUnit test (Instrumented unit test)

About Screen

  • About screen is implemented using web view
